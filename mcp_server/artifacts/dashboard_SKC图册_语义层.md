# 仪表板语义层说明 — 1.2 单品图册(SKC版)

> 版本：2026-05-16
> 仪表板 page_id：`c38a7adbd12bc40eeaa00526`
> 用途：供飞书 Agent 了解仪表板结构、字段含义及数据格式，以便准确回答用户问题。

---

## 一、仪表板概述

本仪表板以 **SKC（最小颜色款式单元）** 为核心粒度，展示各 SKC 的销售、库存、消化率、周转等核心运营指标，覆盖线下（自营/托管）、线上（电商/港澳）全渠道。

**适合回答的问题举例：**
- 某个 SKC 上周卖了多少？折扣是多少？
- 这个 SKC 现在库存有多少，周转几周？
- 某波段/品类的消化率整体如何？
- 某个 SKC 的电商各平台销量拆分是多少？

## 二、可用筛选维度

查询时可通过以下维度进行筛选（对应 `get_card_data` 的 `filters` 参数）：

| 筛选维度 | 说明 | 枚举值 / 示例值 |
|---|---|---|
| `skc编码` | SKC 唯一编码，精确匹配单个 SKC | `6M2JJ44500F37`（各不相同） |
| `实际波段` | 商品实际所在波段（运营排期） | `1A-1`、`1A-2`、`1E`、`2A-1`、`3E`、`3S`、`4A-1`、`4A-2`、`4E-1`、`4E-2`、`5A-1`、`5A-2`、`5E-1`、`5E-2`、`8E`（当季波段，以实际仪表板为准） |
| `运营中类` | 商品运营中类分类 | `牛仔裤`（本仪表板以牛仔品类为主）|
| `运营大类` | 商品运营大类分类 | `下装`（本仪表板以下装为主） |
| `运营系列(新)` | 运营系列 | `DENIM GRID`、`FASHION DENIM GRID` |
| `产品线(新)` | 产品线 | `MAIN LINE`、`EVERYDAY`、`OUTLET`、`BLACK GOLD` |
| `商品年` | 商品所属年份 | `2024`、`2025`、`2026` |
| `商品期` | 商品所属季节/期别 | `春夏`（SS）、`秋冬`（AW） |
| `实际上市日期` | SKC 实际上市日期 | `2026-05-14`（YYYY-MM-DD 格式） |
| `是否上市` | 是否已上市 | `是`、`否` |
| `spu` | SPU 编码（款式级，不含颜色） | `6M2JJ44500`（各不相同） |
| `商品标签` | 商品特殊面料/工艺标注 | `汉麻牛仔`、`麻棉牛仔`、`国产/进口`（部分 SKC 有标注，多数为空） |
| `denim fit-name` | 牛仔版型名称 | `ALICE`、`ANNI`、`BELLA`、`BETTIE`、`BLUE ATTACK`、`FLARES`、`GLENDA PLUS 5`、`JASMINE`、`LOOSE`、`SLIM`、`中裤`、`短裤` |
| `denim fit-family` | 牛仔版型系列（fit-name 的上一级分类） | — |

> **注意**：不传 `filters` 时返回全仪表板聚合数据。

## 五、数据格式规范

### 5.1 数值类型约定

| 类型 | 原始示例 | 展示格式 | 说明 |
|---|---|---|---|
| 销售金额 | `712286`（int） | `71.2 万` | 元为单位，≥10000 换算为万 |
| 销售件数 | `391`（int） | `391 件` | 直接显示 |
| 折扣率 | `0.9154`（float） | `91.5%` 或 `9.2 折` | 乘以 100 显示 |
| 消化率 | `0.1862`（float） | `18.6%` | 乘以 100 显示 |
| 环比变化 | `null` | `—` 或"暂无数据" | 新品首周无上上周数据 |
| 周转周数 | `4.37`（float） | `4.4 周` | 单位是周，**不是百分比** |
| 消化率目标 | `"0.080000000000"`（str） | `8.0%` | 字符串，需先转 float 再格式化 |

### 5.2 富文本字段约定

横排 PIVOT_TABLE 中，多渠道/多维度数据压缩在单个字段值内：
- `\n` 分隔不同维度行
- `/` 分隔同维度内各渠道或区域
- `NULL%` 表示无数据（如新品首周无环比）
- 字段名中的 `<br>` 是表头换行符，解析字段名时可替换为空格或 `/`

### 5.3 渠道缩写对照

| 缩写 | 全称 |
|---|---|
| 自营 | 直营门店 |
| 托管 | 加盟/托管门店 |
| 电商 | 线上电商平台 |
| 港澳 | 港澳门店 |
| 总仓 | 中央仓库 |
| N/E/S/W | 北区/东区/南区/西区 |

## 六、典型查询场景

### 场景 1：按 SKC 查询完整数据

**步骤：**
1. 调用 `get_cards_by_filter("skc编码")` → 获得 7 张响应卡片
2. 对 7 张卡片分别调用 `get_card_data(card_id, filters={"skc编码": "<SKC编码>"})`
3. 汇总展示

### 场景 2：按波段查询汇总

**步骤：**
1. 调用 `get_cards_by_filter("实际波段")` → 同样返回 7 张卡片（与 skc编码 相同）
2. 调用 `get_card_data(card_id, filters={"实际波段": "SS26"})`

### 场景 3：按品类查询

```python
filters = {"运营中类": "牛仔裤"}
# 或叠加多个条件
filters = {"运营中类": "牛仔裤", "实际波段": "SS26"}
```

### 场景 4：查询竖排图册（按品类/商品期）

竖排卡片（`beba57f768e164e699daec5d`）**不响应 skc编码**，适合按品类浏览：
```python
get_card_data("beba57f768e164e699daec5d", filters={"运营中类": "牛仔裤"})
```

## 七、Agent 使用建议

1. **查 SKC 数据**：优先用 `get_cards_by_filter("skc编码")` 找到相关卡片，再逐卡取数。
2. **识别筛选字段**：用户说的词对应的筛选字段参考本文档第二章「枚举值」列。例如「汉麻牛仔」→ `商品标签`；「MAIN LINE」→ `产品线(新)`；「ALICE」→ `denim fit-name`；不要反复用错误字段试探，直接查表匹配。
3. **数值格式化**：`float` 类型字段若字段名含"率"、"折扣"、"▲"，乘以 100 显示为百分比；`周转W` 是周数直接显示。
4. **富文本解读**：横排 PIVOT_TABLE 的多渠道字段值含 `\n` 和 `/`，展示时可按行分段显示，提升可读性。
5. **null 值处理**：新品首周环比字段（`▲销额%` 等）返回 null，属正常业务场景，显示为"首周上市，暂无环比"。
6. **销售金额单位**：原始值为元（整数），显示时换算为万元（保留 1 位小数）。
